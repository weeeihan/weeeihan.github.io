  • Debtap

    React | CSS | Next | Firebase | Supabase | Mathematics (?)

    Handle your debts with just taps! It is never easy to keep track on how much money you owe to others or others to you when you live with a group of friends. This is a much simpler and easy-to-use version of Splitwise, it focuses on ultra-simplicity and friendliness in UI. You log in to see a list of people with unsettled debts, green means they need to pay you, red means you need to pay them. You click into a name to see a list of debts with descriptions. Just add a new debt by keying in the amount and description, select either "Lend" or "Borrow", then we will do the calculations for you! There are two modes in this application - Personal and Sharing. Personal mode is an offline debt recording service, you can add people without linking any account.


    The Sharing mode consists of linked friends and group functions. In this mode, accounts must be created. Linked friends can be done by inserting their emails when adding people into the main page list. The group function is a WIP which helps settle debts within a group, this function will reduce transaction paths to the shortest.     VHDL | Arduino | Raspberry Pi | Matlab | PLC

    Just to remind you again, I do not study software engineering, nor computer science. The three projects I mentioned above are not what my lecturers want us to do, or be good at. Those projects show that I am a versatile learner and a self-taught software adventurer, I am willing to learn things outside of my comfort scope, and try my best to excel at it. However, we do have our own software projects in the EE engineering world.

    The first project I will show you is a VHDL (very high-speed integrated circuit hardware description language) project. This is a temperature sensor, with the temperature reading displayed on seven-segment displays. Sounds pretty easy right? Except when you have to program the whole controller yourself. We were given an empty FPGA (Field Programmable Gate Arrays), which are basically just computers but faster and simpler (and more difficult to deal with). VHDL is the language to program this FPGA.


    Yes, it's just a group of numbers. However, the work behind this 6 numbers is just insane. We have to program every component that leads to these numbers. From obtaining raw binary data from the sensor bit by bit through SPI (Serial Peripheral Interface), to converting the 16-bit binary data into 6 binary coded decimals (BCDs), to store values temporarily in a memory block, to selecting which LEDs on a seven-segment display to light up.

    The second project would be a 4-wheel robot that has the ability to follow lines of different colors, measure ramp angle, measure distance, recognize colors and symbols. Both Arduino and Raspberry Pi controllers were used for this project. The Arduino was programmed in C++ to control the movement of the robot, whereas the Raspberry Pi was programmed in Python for the computer vision. in Python. The communication between this two controller was done via I2C.
